# 助手命令指南

AI 创作助手现在支持通过命令直接进行交互和触发特定工作流。

## 可用命令

### `/help`
显示所有可用的助手命令及其说明。

**示例**：
\`/help\`

### `/review`
使用特定的角色配置对小说当前内容进行深度审校，并将结果提取为结构化“建议”卡片。

**语法**：
\`/review [profile] [text...]\`

*   **profile**: 审校角色，可选值：
    *   `consistency` (默认): 一致性审查员，关注世界观、设定的逻辑冲突。
    *   `quality`: 质量评估员，关注文笔、对话和悬念设置。
    *   `editor`: 主编，关注商业价值、节奏和读者期待感。
*   **text**: 可选。若不提供，则默认对最近编辑的章节进行审校。若提供，则对该自定义文本进行审校。

**示例**：
\`/review\` -> 使用默认一致性角色审校当前章节。
\`/review quality\` -> 使用质量评估角色审校当前章节。
\`/review editor 这是一个段落的测试...\` -> 使用主编角色审校指定段落。

## 动作按钮与快捷指令

助手会在交互过程中根据分析情况自动附带可点击的动作按钮（例如创建人物）。点击按钮内部其实也是通过注册好的动作类型（Action Registry）进行调用的。用户通过UI点击即可完成对应操作。
