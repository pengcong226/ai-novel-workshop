# AI小说工坊 - 调试系统优化清单

基于对项目当前调试接口的综合评价，制定以下优化和重构清单，旨在提升工程化水平、降低调试成本，并清理历史包袱。

## 1. 历史日志系统迁移 (Console Log Migration)
目前项目中存在 200+ 处的 `console.log`，需要逐步迁移至刚建立的统一 `logger.ts` 体系。
*   [ ] **提取与替换**: 将 `src/utils/` 下的核心模块（如 `vectorStore`, `conflictDetector`, `novelImporter`, `llm/` 目录）的控制台打印替换为 `getLogger('模块名')`。
*   [ ] **插件系统日志规范化**: 将 `src/plugins/` 下的大量状态日志（插件加载、注册、生命周期）替换为 `getLogger('plugin')`。
*   [ ] **区分日志级别**: 仔细甄别原先的 `console.log`，将其合理归类为 `debug`, `info`, `warn` 或 `error`。

## 2. 可视化开发者控制台 (Developer UI Panel)
虽然 `window.__APP_LOGGER__` 等提供了控制台调用方式，但缺乏直观的 UI 面板。
*   [ ] **开发模式悬浮面板**: 在 `App.vue` 中封装一个仅在开发模式 (`import.meta.env.DEV`) 或通过特定快捷键呼出的 DevTools 组件。
*   [ ] **日志实时查看UI**: 在面板中提供实时日志输出界面，支持按命名空间和级别过滤日志。
*   [ ] **系统状态监控**: 提供按钮在面板内直接触发 `runConflictDetectionTest()` 或查看 IndexedDB 核心统计信息。

## 3. 离线/免 Token 调试机制 (Mock AI Response)
为了在 UI 开发或核心逻辑测试中避免消耗真实的 AI Token 并加快响应速度。
*   [ ] **模拟 API Provider**: 在 `AIProvider` 注册表中，添加或完善一个内部的 `MockProvider`，拦截请求并快速返回固定的 JSON 或大纲字符串。
*   [ ] **提供切换开关**: 在开发者面板中提供一键切换 "真实 API / Mock 数据" 的功能。

---
*注：优化任务将分阶段进行，确保不影响核心生成业务。*
