# AI小说工坊 - 核心技术难点攻关总结

## 核心问题：100万字长篇小说的上下文管理

### 问题分析

**GPT-4/Claude上下文窗口：约128K-200K tokens**

100万字小说的上下文需求：
- 完整内容：约700,000 tokens（远超窗口限制）
- 最近10章：约35,000 tokens（可接受）
- 最近3章完整+10章摘要+检索：约20,000 tokens（最优方案）

### 解决方案：分层记忆系统

```
┌─────────────────────────────────────────────────────────────┐
│                     Token预算分配                            │
├─────────────────────────────────────────────────────────────┤
│  系统提示 + 世界观核心          1,500 tokens (固定)         │
│  当前章节大纲                   300 tokens (固定)           │
│  活跃人物信息                   1,000 tokens (动态)         │
│  短期记忆（3章）               12,000 tokens (最大50%)      │
│  中期记忆（10章摘要）           5,000 tokens (动态)         │
│  长期记忆（向量检索）           3,000 tokens (动态)         │
├─────────────────────────────────────────────────────────────┤
│  总计                         约20,000 tokens               │
│  剩余给生成                   约4,000-8,000 tokens          │
└─────────────────────────────────────────────────────────────┘
```

## 关键技术点

### 1. 向量检索实现

**选型：ChromaDB**
- 理由：轻量级、Python原生、本地存储、无需外部服务
- 替代方案：FAISS（更高性能，但需手动管理元数据）

**嵌入模型：**
```
本地优先：sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2
- 维度：384
- 速度：快
- 中文支持：良好

在线备选：OpenAI text-embedding-3-small
- 维度：1536
- 精度：更高
- 成本：需API调用
```

**向量化内容分类：**
| 类型 | 重要性 | 检索优先级 | 示例 |
|------|--------|-----------|------|
| 关键事件 | 高 | 优先 | 主角做出重大决定 |
| 人物时刻 | 高 | 优先 | 情感转变、成长 |
| 伏笔 | 高 | 优先 | 线索埋设、暗示 |
| 世界观规则 | 中 | 正常 | 魔法系统规则 |
| 章节摘要 | 中 | 正常 | 每章剧情概要 |
| 重要对话 | 低 | 补充 | 关键对话片段 |

### 2. 记忆更新流程

```
章节完成 → 提取关键信息 → 生成摘要
              ↓
    ┌─────────┴─────────┐
    ↓                   ↓
更新短期记忆         向量化存储
(滑动窗口)           (长期记忆)
    ↓                   ↓
移出最旧章节        更新人物出场记录
    ↓
转为摘要加入中期记忆
    ↓
移出最旧摘要
    ↓
向量化存入长期记忆
```

### 3. 一致性检查机制

**检查时机：**
1. 生成前检查 - 发现潜在问题
2. 生成后检查 - 验证内容一致性

**检查维度：**
- 人物一致性：位置、状态、性格、能力
- 世界观一致性：规则、设定、限制
- 时间线一致性：事件顺序、时间跨度
- 伏笔状态：未回收伏笔数量、时长

### 4. Token优化策略

**动态上下文组装：**
```python
def assemble_context(budget=20000):
    context = []

    # 必需项（固定消耗）
    context += system_prompt      # 500 tokens
    context += worldview_core     # 1000 tokens
    context += chapter_outline    # 300 tokens

    # 动态项（按优先级）
    remaining = budget - 1800

    # 优先级1：活跃人物（预算20%）
    chars = get_characters(max_tokens=remaining*0.2)
    context += chars
    remaining -= count_tokens(chars)

    # 优先级2：短期记忆（预算50%）
    short_term = get_short_term(max_tokens=min(12000, remaining*0.5))
    context += short_term
    remaining -= count_tokens(short_term)

    # 优先级3：中期记忆（预算30%）
    medium_term = get_medium_term(max_tokens=remaining*0.3)
    context += medium_term
    remaining -= count_tokens(medium_term)

    # 优先级4：长期记忆（剩余预算）
    long_term = vector_search(max_tokens=remaining)

    return context
```

## 技术债务与风险

### 已知风险

1. **嵌入模型质量**
   - 风险：本地模型对中文小说语义理解可能不够精准
   - 缓解：提供在线模型选项，支持切换

2. **关键信息提取准确性**
   - 风险：LLM可能遗漏或过度提取关键信息
   - 缓解：多轮校验、人工审核接口

3. **一致性检查误报**
   - 风险：自动检查可能产生误报
   - 缓解：分级警告（error/warning/info）

4. **存储空间增长**
   - 风险：长期运行后向量库可能很大
   - 缓解：定期清理低重要性向量

### 技术债务

1. **摘要质量评估**
   - 需要建立摘要质量评估机制
   - 可能需要人工标注训练数据

2. **人物关系图谱**
   - 当前设计未包含完整的关系图谱
   - 可能需要引入图数据库

3. **多线叙事支持**
   - 当前设计假设线性叙事
   - 多线叙事需要更复杂的上下文管理
