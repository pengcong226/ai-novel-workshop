时间线编辑器功能测试

功能概览

事件提取 已完成

从大纲和章节内容自动提取事件，包括主线剧情、支线剧情、伏笔、人物出场等。

多轨道显示 已完成

支持主线、支线、人物、伏笔等多轨道显示，可自定义显示/隐藏轨道。

事件颜色区分 已完成

不同类型事件使用不同颜色标识，便于快速识别。

缩放和平移 已完成

支持时间线缩放和平移，方便查看不同时间范围的事件。

拖拽调整 已完成

支持拖拽调整事件的时间位置，实时更新章节信息。

事件编辑 已完成

点击事件查看详情，双击编辑事件，右键添加新事件。

冲突检测 已完成

自动检测事件重叠、时间顺序矛盾、伏笔未揭示等冲突。

导出图片 已完成

支持将时间线导出为PNG图片，方便分享和保存。

使用说明

1. 访问项目编辑器，点击左侧菜单"时间线"
2. 点击"提取事件"按钮自动从大纲和章节提取事件
3. 使用复选框控制显示的轨道和事件类型
4. 点击事件查看详情，双击编辑
5. 右键时间线空白处添加新事件
6. 拖拽事件调整时间位置
7. 点击"检测冲突"检查时间线问题
8. 点击"导出图片"保存时间线截图

事件类型说明

主线事件 (蓝色)

主要剧情线，包括主线剧情、卷大纲、主要事件等。

支线事件 (绿色)

支线剧情，独立于主线但与主线相关的副线故事。

人物事件 (橙色)

人物出场、成长、关系变化等与人物相关的事件。

伏笔事件 (灰色)

伏笔埋设和揭示，显示伏笔的持续范围。

自定义事件 (红色)

用户手动添加的自定义事件，灵活记录重要节点。

冲突类型

事件重叠 警告

同一轨道内的事件在时间上存在重叠，可能需要调整。

伏笔未揭示 警告

已埋设的伏笔尚未揭示，提醒作者记得回收伏笔。

时间矛盾 错误

事件的开始和结束时间存在逻辑矛盾，需要修正。

技术实现

// 文件结构
src/
├── components/
│   └── TimelineEditor.vue        # 时间线编辑器组件
├── utils/
│   └── timelineExtractor.ts      # 事件提取和冲突检测
└── views/
    └── ProjectEditor.vue         # 集成时间线菜单项

// 主要功能
- extractTimelineEvents()         # 从项目提取事件
- generateTimelineGroups()        # 生成分组/轨道
- detectTimelineConflicts()       # 检测时间线冲突
- exportTimelineAsImage()         # 导出时间线为图片

// 使用的技术栈
- vis-timeline: 时间线可视化库
- html2canvas: 截图导出
- Element Plus: UI组件

性能优化建议

  • 当事件数量超过500个时，建议使用轨道过滤功能
  • 大量事件时，使用缩放功能查看特定时间范围
  • 定期清理已完成或不需要的事件
  • 导出图片前建议先过滤显示轨道，减少渲染负担
